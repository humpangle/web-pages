SQL cheat sheet

Chachrayatin

Jun 24, 2020·9 min read

This cheat sheet covers all concepts from novice to advanced level

Structured Query Language (SQL) is widely being used as a Relational Database Programming Language almost everywhere from a small project on a school kid to a large enterprise. Hence, achieving mastery in the SQL adds a bonus to your skillset.

Here, I will be discussing some important functions and concepts of SQL which will definitely add a star to your skills.

Date Conversion Functions

Add: function to add some interval to your timestamp column

Syntax: date_add(column_name,interval units value)
where units: Integer value we want to add
value: Minute/Hour/Second/Day/Month/Year
Example: select date_add(current_timestamp,interval 330 minute)

Difference: function for comparison of two timestamps

Syntax: timestampdiff(value,column1,column2)
where units: Integer value we want to add
value: Minute/Hour/Second/Day/Month/Year
Example: select timestampdiff(hour,current_timestamp,registered_at)

Subtract: function to subtract some interval from your timestamp column

Syntax: datesub(column_name,interval units value)
where units: Integer value we want to add
value: Minute/Hour/Second/Day/Month/Year
Example: select date_sub(current_timestamp,interval 330 minute)

These functions are highly helpful in Timestamp conversions like from UTC to IST and also for calculating the difference between two timestamps like the difference in days from registration to transaction date

Conditional Functions

If-Else: Just like any other programming language SQL also supports if-else case handling.

Syntax: if(condition,true case,false case)

Example: select if(gender=’m’,’MALE',’FEMALE’) from students

Switch Case: Just like any other programming language SQL also supports switch case handling.

Syntax:
 case
when condition then do
when condition then do
else do
end

Example:
select case
when gender=’M’ then ‘MALE’
when gender=’F’ then ‘FEMALE’
else ‘UNKNOWN’
from user;

Aggregate Functions

sum(), avg(), count(column_name), count(*) are some popular aggregate functions.

Important use case of sum():
Let’s think of example we have to count number of male and female students from our Students Table.
select sum(if(gender=’M’,’1’,’0’)) as male_count,
sum(if(gender=’F’,’1',’0')) as female_count
from user;

Important Concepts

  1. Join:
    As the name suggests it is for combining the two or more tables together on the same matching column.
    There are four types of join and out of these four, 2 are really important to understand.

a. Inner Join or Join: The resultant table after join will only have the rows which are mutual to the tables on which joined is applied.

Syntax:
select *
from students s join student_grades sg on sg.student_id = s.student_id;

b. Left Join: The resultant table will have all the rows of the main table but only matching rows of the secondary table.

Syntax:
select *
from students s left join student_grades sg on sg.student_id = s.student_id;

c. Right Join: The resultant table will have all the rows of the secondary table but only matching rows of the main table.

Syntax:
select *
from students s right join student_grades sg on sg.student_id = s.student_id;

d. Full Join: The resultant table will have all the rows of the secondary table and main table both there will be both the matched and unmatched rows.

Syntax:
select *
from students s full join student_grades sg on sg.student_id = s.student_id;

Important: If the column on which join is made is having repeated values then the resultant table will also have repeated rows as “join” has nothing to do with repetition. It will add data as many times the same column is found

Note: we can join 2,3,4…N number of tables together :)

2. Group:
Consider grouping as creating a cluster on a similar type of data or to be more precise on a column of our data. Like if we consider the definition of a class which is “A class is a group of similar kind of Objects”.

How to know when we have to apply group on a column?

a. When we need a data like “each attribute”
Example: Photos Uploaded on each date, Registrations on each date

b. When we need a data like “per attribute”
Example: Transactions per user, Revenue per user

Important: On using group we usually also use the aggregate functions and to place conditions on the aggregate column. So to put conditions on that aggregate column we can't use “Where” we have to use “Having”.
This is due to the execution order of the query which is like “where” is executed before “select” and “group by” and due to which at the time of “where” execution the aggregate column does not exist which leads to error.

3. Union & Union All:
This is a rarely used operation in SQL but it is important in solving some highly complex queries or demands.
What join does is, it attaches columns horizontally.
What union does is, it attaches columns vertically.

If we want to count the students involved in conflicts or the number of times a student was involved in a conflict. We can use group by and count(*) but to use it we want both the columns in one column like:

Syntax:
select student_1 as student_id
from student_conflicts
union
select student_2 as student_id
from student_conflicts

Now, we can easily apply the group by and count(*) methods.

Note: When we use Union, all the values will be present in the resultant column but only once if they are repeated and if we use “union all” we will get all the values in the same number they were present in the two columns

4. Inner Query:
One of the most complex and most important concept to understand. Many developers fail to build inner queries.

How to know when we have to use the inner query?
It is required when we a combined or collective result from a table but we also have to perform some operations before getting the results.

Example: We have student_conflicts table as described above now we have to make a query to know how many times a student is involved in a conflict.

Query:
select student_id,count_conflicts
from students
left join (select student_id, count(*) as count_conflicts
(select student_1 as student_id
from student_conflicts
union
select student_2 as student_id
from student_conflicts)t1
group by student_id)

Explanations: Here we were unable to get the count of conflicts directly from the student_conflicts table so we calculated it separately by creating a temporary table and then computed the results.

It is an important concept that you should know what inner queries are but you must use these queries wisely because it creates performance issues when used on a huge database. So i always recommend to run such queries on a limited dataset via limit of SQL and step by step increase limit.

In many cases the inner query can be replaced with a Join. But in the case when you need to perform some aggreagations before using JOIN we need an Inner Query.

5. In & Not In:
In and Not in both consider having some temporary table to be given in the brackets. These can be used to check whether some column value of one table exists in other tables or not.

Example:
select * from students where student_id not in (select user_id from alumni);
select * from students where student_id in (select user_id from alumni);

6. Exists & Not Exists:
Exists and Not exists both have the same usage as the In and Not In respectively.

Example:
select * from students where not exists (select true from alumni where alumni.student_id=students.student_id);

select * from students where exists (select true from alumni where alumni.student_id=students.student_id);

Note: The difference between (In,Not In) and (Exists,Not Exists)is of the performance. In-Not In creates another temporary table or nested query which degrades the time complexity of the query

The time complexity of your Query

Using Index:
Defining Index to the columns is a must from a developer point of view as it will definitely improve the speed of the query. An index should be placed on the columns which will be frequently used for searching the data like user_id or timestamp.
Index means how the data inside the table will be stored while processing a query. Indexing is done by multiple DataStructure.
Most popularly used index is B-Tree.

We can relate time complexity these Database Index data structures with the real world Datastructure Algorithms.
Like B-Tree will do operations in O(logN)

To view the index of your table use:

show index from students;
and check the columns index_type to know the index algorithm used and sequence_index to see the columns having index.

It is important for a developer to create a query which uses an index as it will improve the execution time of the query.
We can check whether the query is using an index or not by using “describe”:

describe select * from students where id=12;
and check the using column if it shows “index” means query designed is using the index and if it shows “where” means query is not using the index.

Order of your Query Execution

  1. FROM, including JOINs
  2. WHERE
  3. GROUP BY
  4. HAVING
  5. WINDOW functions
  6. SELECT
  7. DISTINCT
  8. UNION
  9. ORDER BY
  10. LIMIT and OFFSET

Where V/S Having:
On using group we usually also use the aggregate functions and to place conditions on the aggregate column. So to put conditions on that aggregate column we can not use “Where” we have to use “Having”.
This is due to the execution order of the query which is like “where” is executed before “select” and “group by” and due to which at the time of “where” execution the aggregate column does not exist which leads to error.

Know your Database & Tables

To know the list of all the tables that exist in your database use:

Syntax: show tables;

Knowing your table is an important aspect to build a query in order to see the datatypes of the column and various properties associated to a column like Nullable, Default Values, Primary Key can be seen using:

Syntax: desc table_name;

Important Tips

1. Set primary key to Id

Don’t set user_id or student_id as the primary key as at one instance of time or another, Id will be needed as the primary key. It is a good practise to make id as primary key and set user_id or student_id as UNIQUE.

2. Using Index:

Set Index to the columns which will be frequently used in your query. This will improve the time complexity of your query. Moreover, use “describe” to verify if the indexing is being used in your query or not. “Describe” is already explained above.

3. Timestamps Column:
Do make three additional columns in your table which are :

created_at: values will be inserted here only once at the time of row creation.

updated_at: values in this column must be updated each time when we edit the row. At time of row creation timestamps are added to here also.

deleted_at: values will be inserted to this when we wish to delete the row. So, we will not delete the row but we will insert the timestamp to this column which will tell us that the row is deleted. So for the rows having null values in this column are still the live rows.

4. Assign column datatype size according to requirements only. Like, try to analyze what will the maximum number of characters or maximum number have to be stored to the column then define the size. The too-small size of the column datatype will definitely give an error in future and too large size of the column will result in memory wastage.

5. Always use the inner query with some limit of data.

6. Last but this is quite important, It’s about the naming of the table according to the best practices. The rule is the table name is always plural like for a table with data of students of a school name should be Students but not the student. As each row of Students Table will be a student.
